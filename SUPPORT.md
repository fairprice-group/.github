# Support

## Getting Help

Each repository is maintained by its own team. For questions, bugs, or feature
requests, please use the **GitHub Issues** on the relevant repository.

If you're unsure which repository to raise an issue in, check the repository's
README.

## Before Asking for Help

1. **Search existing issues** — your question may already be answered
2. **Check the documentation** — READMEs, wikis, and internal docs
3. **Try to reproduce** — gather error messages, logs, and steps to reproduce

## Reporting Bugs

Use the repository's bug report template when filing issues. Include:

- Clear description of the problem
- Steps to reproduce
- Expected vs actual behaviour
- Environment details (OS, runtime version, etc.)

## Feature Requests

Use the repository's feature request template. Describe:

- The problem you're trying to solve
- Your proposed solution
- Any alternatives you've considered

## Security Issues

For security vulnerabilities, **do not open a public issue**. See
[SECURITY.md](SECURITY.md) for responsible disclosure instructions.
